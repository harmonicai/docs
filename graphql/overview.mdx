The Harmonic GraphQL API lets you request exactly the data you need, reducing response payload size and improving performance.

## Endpoint

```
POST https://api.harmonic.ai/graphql
```

## Authentication

Pass your API key via the `apikey` header, the same as the REST API:

```bash
curl -X POST 'https://api.harmonic.ai/graphql' \
  -H 'Content-Type: application/json' \
  -H 'apikey: your_api_key' \
  -d '{"query": "{ getCompanyById(id: 1) { name domain } }"}'
```

## GraphQL Explorer

The Harmonic Console includes an interactive GraphQL Explorer where you can build and test queries against the full schema. Access it at [console.harmonic.ai](https://console.harmonic.ai).

The explorer provides:

- **Auto-complete** for fields and types
- **Schema documentation** built into the sidebar
- **Query history** so you can revisit past queries

## Example queries

### Get a company profile

```graphql
query GetCompany {
  getCompanyById(id: 12345) {
    name
    domain
    description
    foundedDate
    headcount
    funding {
      totalFunding
      lastRoundDate
      lastRoundType
      lastRoundAmount
    }
  }
}
```

### Get a person profile

```graphql
query GetPerson {
  getPersonById(id: 67890) {
    name
    linkedinUrl
    currentTitle
    currentCompany {
      name
      domain
    }
  }
}
```

### Enrich a company (mutation)

```graphql
mutation EnrichCompany {
  enrichCompany(input: { websiteDomain: "example.com" }) {
    id
    name
    domain
    enrichmentStatus
  }
}
```

### Paginate through results

GraphQL queries use cursor-based pagination. Pass `first` for page size and `after` for the cursor:

```graphql
query GetPeopleInNetwork {
  getPeopleInNetwork(first: 25, after: "cursor_from_previous_page") {
    edges {
      node {
        name
        linkedinUrl
      }
    }
    pageInfo {
      hasNextPage
      endCursor
    }
  }
}
```

## GraphQL-only features

Some features are only available through GraphQL:

- [**Deal Data — Investors and Financing**](/api-reference/deal-data/get-investor-by-urn) — Investor profiles, investments, and co-investors (paid add-on)
- [**Network Mapping — People in network**](/api-reference/network/get-people-in-network) — People connected to your team
- [**Network Mapping — Companies in network**](/api-reference/network/get-companies-in-network) — Companies connected to your team
- [**People List Updates**](/api-reference/people-lists/get-people-updates) — Recent changes to people watchlist entries
- [**Company Global Field Values**](/api-reference/company-global-fields/get-global-field-values) — Read custom global field values on companies
- [**Person Global Field Values**](/api-reference/person-global-fields/get-global-field-values) — Read custom global field values on people

## Pagination in GraphQL

GraphQL queries use cursor-based pagination with `first`/`after` parameters:

| Parameter | Description |
|-----------|-------------|
| `first` | Number of results to return |
| `after` | Cursor from the previous page's `pageInfo.endCursor` |

The response includes a `pageInfo` object:

| Field | Description |
|-------|-------------|
| `hasNextPage` | Whether more results are available |
| `endCursor` | Cursor for fetching the next page |

## Rate limits

The GraphQL endpoint shares the same [rate limits](/guides/rate-limits) as the REST API: 10 requests per second.
